# Product Planning Workspace

This folder is the shared product strategy and execution workspace.

Use it as the project memory that bridges:

- messy product input
- structured product artifacts
- implementation reality
- QA and handoff context

## Core Files

- `product-specs.md`: living record of what is actually implemented/in progress
- `decision-log.md`: key product decisions and scope changes
- `session-handoff.md`: latest cross-tool handoff snapshot (Claude Code/Codex/Cowork)

## Subfolders

- `audits/`: product audits and assessments
- `brainstorms/`: messy-to-structured ideation outputs
- `proposals/`: solution proposals and tradeoffs
- `prds/`: approved and draft PRDs
- `diagrams/`: flow, lifecycle, and system diagrams
- `plans/`: implementation plans and phases
- `feedback/`: QA notes, review notes, and change requests
- `handoff/`: PM handoff notes and release-facing summaries

## Working Rules

- Keep artifacts functional and PM-readable.
- Treat PRDs as intended behavior.
- Treat `product-specs.md` as actual behavior.
- If implementation changes the feature, update `product-specs.md`.
- Keep scope and backlog boundaries explicit.
